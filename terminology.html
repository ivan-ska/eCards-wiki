Терминология сервиса

Общие понятия

2FA

Двухфакторная аутентификация — дополнительный уровень защиты аккаунта. После ввода пароля требуется подтвердить вход одноразовым кодом из приложения-аутентификатора. Настраивается в Мой аккаунт → Безопасность.

Decline Rate

Процент отклонённых операций. Влияет на размер штрафа за каждый деклайн. Рассчитывается при наличии более 100 операций:

(деклайны / операции) × 100

Если Decline Rate превышает 50% — размер штрафа может быть увеличен, аккаунт может быть заблокирован. Текущее значение отображается в Мой аккаунт → Главная.

Реферал

Новый пользователь, зарегистрировавшийся по вашей реферальной ссылке. Вы получаете 0,25% со всех его платежей в течение 6 месяцев с момента регистрации.

Счета

Базовые счета

Счета для зачисления внешних средств (пополнений). В кабинете доступны два: в USD и EUR. Пополнение — без комиссии. С базового счёта средства переводятся на счета карт.

Лимитные счета

Счета, на которых у каждой карты есть собственный баланс (лимит). Средства зачисляются на конкретную карту, а не на общий пул.

Единые лимитные счета

Особый тип лимитного счёта, с которого можно выпускать карты разных БИНов (в отличие от обычного лимитного, привязанного к одному БИНу). Делегируется команде. Owner пополняет счёт — сотрудники выпускают на него карты и тратят общий баланс, каждый в своём кабинете.

Счета с общим балансом

Счёт, где все карты тратят из единого пула средств без индивидуальных лимитов. Пополняется общий счёт, а не отдельные карты. Пока доступен только для EUR-BIN.

Карты и лимиты

Лимитные карты

Карты с индивидуальным балансом. Пополняются через увеличение лимита. При выпуске можно задать стартовый баланс.

Карты с общим балансом

Карты, выпущенные на счёте с общим балансом. Не имеют индивидуального лимита — тратят из общего пула счёта.

Постоянные лимиты

Фиксированный лимит на карте, не сбрасывающийся автоматически. Устанавливается вручную через контекстное меню карты → «Изменить постоянный лимит».

Циклические лимиты

Лимиты, автоматически сбрасывающиеся по истечении периода: день / неделя / месяц. Период отсчитывается от момента совершения транзакции, а не по календарю. Доступность зависит от BIN — уточняйте у поддержки.

Команды и делегирование

Команды и роли

Команда — группа сотрудников с общим доступом к определённым счетам.

Роли:

  • Owner (Вл.) — владелец аккаунта, полный доступ.
  • Team Lead — руководитель команды. Имеет полный контроль над картами делегированных команде счетов: делегирует карты, изменяет лимиты, закрывает карты, добавляет в группы, редактирует заметки, переводит средства с/на делегированные счета, видит все операции по ним, получает 3DS коды по картам и уведомления с подтверждением действий фармеров.
  • Buyer — медиабайер, работает с картами в рамках делегированных счетов. Может передавать карты коллегам, видит свою команду и список коллег.
  • Farmer — работает с картами делегированного счёта, но все действия (выпуск и закрытие карт, изменение лимитов, редактирование заметок, добавление в группы) требуют подтверждения Owner. Owner получает уведомления о каждом действии фармера и может подтвердить или отклонить его. Баланс делегированного счёта для фармера скрыт.
  • Финансист — видит все карты и операции по картам/счетам, делегированным команде. Может выгружать отчёты. Не может совершать действия со счетами и картами.

Делегирование

Предоставление сотруднику или команде доступа к счёту. Делегированный счёт отображается у сотрудника в кабинете — он может выпускать на него карты и видеть статистику.

Операции и статистика

Статусы операций по картам

  • Debit (Authorization) — авторизация / холд (платёж инициирован, ожидает списания).
  • Debit (Declined) — операция отклонена.
  • Charge (Refund) — возврат средств на карту.

Статусы операций по счетам

Типы операций в разделе Мои счета → Операции по счетам:

  • Перевод между счетами
  • Изменение лимита карты
  • Комиссия / штраф за деклайн
  • Системные списания

Какие операции входят в статистику

В разделе Статистика трат по картам учитываются фактические расходы по картам: операции типа Debit (Authorization) и Charge (Refund). Деклайны в статистику расходов не входят.

3DS SMS, 3DS пароль, код авторизации

  • 3DS SMS — одноразовый код, приходящий при 3DS-подтверждении платежа. Поступает в Telegram-бот уведомлений.
  • 3DS пароль — постоянный код, указанный в реквизитах карты. Запрашивается на некоторых сайтах как второй уровень подтверждения. Одинаковый для всех карт данного BIN.
  • Код авторизации — код, который рекламная платформа (Google Ads и др.) запрашивает при привязке карты. Приходит в Telegram-бот.
